# PRD: ZIASTACK DR 복구 자동화

## 목표
백업 이미지 기반 VM 재해복구 프로세스에서 엔지니어의 수동 개입을 최소화하고, AI 에이전트가 분석·정책 생성·실행·보고를 자동화한다.

## 사용자
- **주 사용자**: ZIASTACK 플랫폼 운영 엔지니어 (복구 요청 접수, 정책 최종 승인)
- **간접 사용자**: 복구 보고서를 받는 관리자 (Slack, Jira)

## 핵심 기능

1. **복구 정책 자동 생성** — AI 에이전트가 OpenStack 리소스 데이터 + RAG(런북/이력)를 조합해 최적 복구 정책을 도출
2. **엔지니어 승인 루프** — 생성된 정책을 UI에 표시, 엔지니어가 승인/거절로 최종 확정
3. **VM 자동 프로비저닝** — 타겟 서버 없을 시 MCP Tool Calling으로 VM 생성 + ZConverter Agent 자동 설치
4. **복구 실행 및 모니터링** — OpenStack SDK 기반 복구 실행, 실시간 상태 스트리밍 (WebSocket)
5. **자동 리포트 + 알림** — 복구 완료/실패 시 AI 리포트 생성, Slack/Jira 자동 발송
6. **지식 누적** — 복구 성공/실패 사례를 Chroma에 축적해 RAG 품질 지속 향상

## MVP 제외 사항
- 멀티 클라우드 지원 (AWS, GCP 등) — OpenStack 단독 우선
- 복구 정책 수동 편집 UI — 승인/거절만 제공, 파라미터 직접 수정 불가
- 예약 복구 스케줄링
- 복구 롤백 자동화

## 디자인
- Django · Horizon 기반 운영자 대시보드 (기존 UI 확장)
- 다크 테마 고정, 운영 도구 느낌 (마케팅 페이지 아님)
- 실시간 상태는 WebSocket 스트리밍으로 표시
